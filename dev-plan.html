核心主題「駭入對方的心，比駭入系統更難。」

1.4 劇情架構

序章（共同路線）
  ├─ 第一章：入隊（4 天，介紹所有角色）
  ├─ 第二章：初戰（好感度分岐點 → 選擇深入了解的對象）
  └─ 好感度達標 → 進入角色專屬路線

角色路線（各 3 章）
  ├─ 第三章：解題羈絆（合作解題，情感升溫）
  ├─ 第四章：裂縫（誤解或衝突，考驗選擇）
  └─ 第五章：決賽夜（依好感度 → 三種結局）

結局分支：
  Good End  （好感 ≥ 80） 告白 + 贏得比賽
  Normal End（好感 50-79）曖昧收場，留白
  Bad End   （好感 < 50） 錯過，獨自離開

2.4 存檔系統（SaveSystem）規格

class SaveSystem {
  static SLOTS = 6;
  static KEY_PREFIX = "ch_save_";

  // djb2 hash — 計算關鍵欄位的防竄改摘要
  static _hash(state) {
    const critical = JSON.stringify({
      affinity:      state.affinity,
      clearedRoutes: state.clearedRoutes,
      unlockedCGs:   state.unlockedCGs
    });
    let h = 5381;
    for (let i = 0; i < critical.length; i++) {
      h = ((h << 5) + h) + critical.charCodeAt(i);
      h |= 0;
    }
    return h.toString(16);
  }

  static save(slot, state) {
    const payload = { ...state, _hash: this._hash(state) };
    localStorage.setItem(this.KEY_PREFIX + slot, JSON.stringify(payload));
  }

  static load(slot) {
    const raw = localStorage.getItem(this.KEY_PREFIX + slot);
    if (!raw) return null;
    const { _hash, ...rest } = JSON.parse(raw);
    if (this._hash(rest) !== _hash) {
      console.warn("[SaveSystem] 存檔完整性驗證失敗，可能遭到竄改。");
      return null;
    }
    return rest;
  }
}

2.6 資安實作規範

SECURITY REQUIREMENTS以下規範為強制要求，所有 Phase 均須遵守。
風險規範
XSS所有動態文字（含玩家名字）一律用 element.textContent = …，嚴禁 innerHTML 注入任何外部輸入
存檔竄改localStorage 存檔含 djb2 checksum，載入時驗證，失敗拒絕並顯示警告
原型鏈污染謎題答案比對使用 Map 或 Object.hasOwn()，不用裸 object
敏感資料零 API Key、零外部請求、所有資源本地化
依賴風險零 npm / CDN JS 依賴，降低供應鏈風險
玩家輸入名字 maxlength="6"，讀取時 sanitize 至純文字，禁止 eval / Function 操作

RULES

給 Gemini 的強制規則

MANDATORY以下規則為強制要求。任何一條違反都應立即停止並修正。
🔒 禁止 innerHTML 注入動態內容：所有玩家名稱、對話文字等動態內容，一律使用 element.textContent = …。innerHTML 只能用於靜態模板字串（不含任何外部輸入）。
🔐 存檔 checksum 為必要功能：SaveSystem._hash() 的 djb2 checksum 驗證不可省略，必須在 load() 時驗證，失敗拒絕載入。
📦 零外部 JS 依賴：禁止引入任何 CDN JavaScript（jQuery、lodash、axios 等）。唯一例外是 Google Fonts CSS（非 JS）。
🗂️ 劇本只定義資料：場景 JS 檔案只輸出資料陣列，禁止在資料層操作 DOM 或 localStorage。所有副作用由 Engine 處理。
🔑 所有按鈕 ID 唯一且語意化：依本文件定義的 id 命名規範實作，便於自動化測試。
🖼️ 圖片尚未就緒時使用 CSS 漸層佔位：角色立繪若圖片路徑不存在，用角色色票的漸層 CSS 背景取代，保持介面可運作不報錯。
📝 依 Phase 順序實作，完成後回報：完成每個 Phase 後必須回報「P[N] 完成，已實作：…」再繼續下一個 Phase。
